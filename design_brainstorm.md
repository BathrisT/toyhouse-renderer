# Протокол 0

Название всей соцсети/вселенского мега-аппа — "Протокол 0".

Виды страниц на toyhouse:
1. Страница персонажа
2. Варнинг (предупреждение перед заходом в профиль/страницу персонажа)
3. Папки
4. Профиль персонажа
5. Читалка для лора и подобного

Общие требования для страниц:
1. Адаптивный дизайн, в т.ч. для мобильных устройств
2. Современный дизайн вписывающийся в тойхауз. Он не должен выглядеть совсем уж так будто это айфрейм
3. Акцентный цвет: #00fbff. Остальные цвета нужно будет подобрать самому в процессе. Очевидно, должно выглядеть ествественно и стильно
4. Оформление всех страниц, будто это некая общая соцсеть внутри одного приложения

Общая концепция "Протокол 0" (важно для стилистики ВСЕХ вкладок): по лору это гигантская
соцсеть, распространённая среди множества миров — не просто "лента", а настоящий мега-апп
(банк + соцсеть + куча всего ещё), а-ля WeChat/другие суперапы, но с нормальным,
современным дизайном (не наляписто). Единый визуальный язык нужен для всех вкладок
персонажа; "Досье" — не отдельный дизайн, а лёгкая ре-стилизация той же общей системы под
свою тему (см. секцию "Досье").

Реальная доступная ширина контента на Toyhouse — не такая уж маленькая: в примерах
(`11.html`/`12.html`) встречается внешний контейнер `max-width: 1000px` с многоколоночной
раскладкой внутри. Значит постоянная левая колонка меню технически поместится и на
десктопе без урезания функциональности (подробности — в возможности_и_ограничения.md).

Делаем НЕ под видом мобильного приложения (без рамки телефона/чехла устройства) — это
сайт/веб-версия соцсети "Протокол 0", а не скриншот апки. Стилистика может быть
app-вдохновлённая (шрифты/цвета/карточки), но без литерального "экрана телефона" вокруг.

Будем идти по страницам. 

# Страница персонажа

Должен выглядеть как страница соцсети. Основная логика работает на левом меню (логика табов). Там будут следующие табы + подразделение некоторое этих табов:

1. "Профиль": соцсеть профиль пользователя. Наполнение примерно:
- Шапка профиля
- Аватарка
- Имя
- Статус-тег
- день рождения
- мир, город
- семейный статус
- ... (допродумать че там ещё может быть)
- Список постов идущих сверху вниз (новые сверху). Типы постов: текст, текст+картинка,
  чисто картинка. У каждого поста — декоративный (нерабочий, т.к. нет JS) классический
  соцсетевой функционал: лайк, коммент, репост — просто визуально, без реальной логики
- В шапке профиля нужны стандартные соцсетевые счётчики (подписчики/посты и т.п. —
  что обычно есть в соцсетях). Бейдж верификации ("галочка") — НЕ нужен.

2. "Досье": Краткое: Условно базовая информация в пару абзацах как на примерах. А-ля Петя Петров, работник столовой 20 лет живёт в мире а. Полное: Условно нажимаешь "раскрыть все" И видишь  полную биографию. Огромный текст а-ля родился, вырос, учился, работал, умер. Надо продумать че ещё, но ещё накидываю:
- Имя, мир, день рождения, город, семейный статус, ориентация и так далее, все что может быть в условно в досье спецслужб. Оформление тоже соотвеующее, идея в том что к этому блоку в этой соцсети имеют доступ только некоторые сотрудники спецслужб
3. "Друзья": (это просто ссылка на внешний ресурс, без отдельного рендера)
4. "Внешние данные": Что-то типа медкарты. А-ля рост 170, волосы блонд, одевается так-то так, проколоты уши.
5. "Галерея": (это просто ссылка на внешний ресурс, без отдельного рендера)
6. "Инциденты": (это просто ссылка на внешний ресурс, без отдельного рендера)
7. "Соцсети": страница с различными соцсетями персонажа
   (уточнение: это ссылки на ВНЕШНИЕ платформы — Дискорд, Телеграм и т.д., а не под-аккаунты
   внутри "Протокол 0". На практике вкладка нужна только на странице личного персонажа
   автора, у остальных персонажей её не будет — но компонент проектируем сразу как
   переиспользуемый, для любого персонажа при необходимости)

Любые вкладки должно быть легко скрыть как-то, надо продумать, у разных персонажей может быть не все доступно
(решено: на Toyhouse нет условной логики, поэтому "скрыть вкладку" = руками убрать пункт
меню + соответствующий tab-pane из кода при заведении конкретного персонажа, а не тумблер
для зрителя. ОК, так и задумывалось)

Блок вкладок должны быть sticky

Дизайн-референс блока вкладок +-: C:\PythonProjects\namirha_toyhouse\images\image (9).png
(уточнение: референс = только общий вайб/эстетика приложения-соцсети. Навигация по
вкладкам персонажа = именно ЛЕВОЕ вертикальное меню, горизонтальную строку-переключатель
типов контента (Фото/Музыка/Посты/...) с референса под неё брать НЕ надо)

Список постов в "Профиле" — статичный, руками поддерживаемый список карточек, без реальной
подгрузки/лайков/комментариев (JS нет). Это осознанно, задумано именно так.

"Досье" со стилизацией "доступ только спецслужбам" — чисто визуальная/лором обусловленная
условность, никакого технического ограничения доступа тут физически быть не может (статика,
нет логина/прав). Нужно просто красиво оформить под этот флёр, в рамках общей стилистики
"Протокол 0" (не отдельный дизайн-язык, а лёгкая ре-стилизация: другая палитра/акценты,
может печатная типографика, штампы).

Открытый вопрос — структура "Полное досье": одним блоком "раскрыть всё", или по главам
(отдельные сворачиваемые секции а-ля "chapter 1: детство", "chapter 2: учёба" и т.д., как
аккордеон). Пока склоняюсь ко второму варианту (по главам) — больше похоже на настоящее
досье/дело, лучше масштабируется на длинные биографии, и технически это тот же проверенный
аккордеон-паттерн, ничего сложнее не требует. Но решение пока не финальное.

Всякие повторяющиеся данные, вроде как имя и тд, что в профиле что в досье надо продумать как хранить где-то в одном месте чтобы удобно редачить если это вообще возможно
(идея на будущее: в чистом Toyhouse-HTML единого источника данных нет (нет include/
переменных на стороне сайта). Возможное решение — локальный "билдер"/генератор страниц
(на Python, раз уж это Python-проект): держим данные персонажа в одном месте локально,
шаблонизируем, на выходе получаем готовый плоский HTML для вставки на сайт. Отложено:
сначала прорабатываем сами дизайны страниц вручную, и только если получится сделать
красиво — пишем скрипт-генератор.)
