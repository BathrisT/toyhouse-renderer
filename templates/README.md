# Шаблоны — как это работает

Файлы в этой папке (`templates/*.html`) — не готовые страницы для вставки на Toyhouse.
Это исходники для инструмента `tools/index.html`: он их парсит, строит по ним форму,
и уже готовый (отрендеренный, без плейсхолдеров) кусок HTML можно скопировать или
скачать и вставить в профиль/блог/куда угодно на Toyhouse.

Каждый шаблон обязан быть валидным для вставки на Toyhouse и после рендера, то есть
следовать всем ограничениям площадки — см. `../возможности_и_ограничения.md`
(особенно раздел КРИТИЧНО: WYSIWYG, `gap`/`background`, `<div>` внутри `<a>`,
`.collapse`/`.nav`). Сам HTML шаблона пишется как обычный HTML-фрагмент
(без `<html>/<head>/<body>`), только с плейсхолдерами вместо конкретных значений.

## Синтаксис плейсхолдеров

```
{{поле}}                    — простое поле, однострочный текст
{{поле|textarea}}           — многострочное поле (в форме — <textarea>)
{{поле|url}}                — поле-ссылка (в форме — <input type="url">)
{{поле|number}}              — числовое поле (в форме — <input type="number">)
{{поле|color}}               — выбор цвета: сначала палитра пресетов + кнопка
                                «Другой» (открывает обычный нативный выбор цвета)
{{поле|checkbox}}            — галочка (в форме — <input type="checkbox">). Значение
                                "1" = включено, "" = выключено — этим удобно управлять
                                условными блоками {{#поле}} (см. ниже)

{{поле=значение}}            — РЕАЛЬНЫЙ дефолт: подставляется в форму сразу,
                                можно оставить как есть и оно попадёт в рендер
{{поле~подсказка}}           — ТОЛЬКО подсказка: поле в форме стартует ПУСТЫМ,
                                подсказка показывается серым текстом (placeholder)
                                и никогда не попадёт в рендер сама по себе

{{поле|textarea=значение}}   — тип + реальный дефолт вместе
{{поле|textarea~подсказка}}  — тип + подсказка вместе
{{поле|checkbox=1}}          — галочка, включена по умолчанию

{{#each группа}} ... {{/each}}  — повторяющийся блок, любое число элементов.
                                    Внутри — обычные {{поле}} для полей ОДНОГО элемента.
{{@index}}                   — порядковый номер элемента внутри {{#each}} (считая с 1)
{{../поле}}                  — внутри {{#each}}, обращение к полю из внешнего
                                (родительского) контекста, а не текущего элемента группы

{{#поле}} ... {{/поле}}      — условный блок: рендерится, только если поле непустое
                                (например картинка в посте, которой может не быть).
{{#поле|checkbox=1}} ... {{/поле}}  — то же самое, но открывающий тег ЗАРЕГИСТРИРОВАН
                                как отдельное булево поле формы (галочка) — не нужно
                                ещё раз писать {{поле}} внутри блока. Основной паттерн
                                для «показать/скрыть кусок» и «включено по умолчанию,
                                можно выключить» — см. вкладки в character_page.html
                                (показать/сделать стартовой) и теги в warning.html.
```

HTML-комментарии (`<!-- ... -->`) вырезаются перед разбором шаблона — можно свободно
писать документацию внутри файла, `{{...}}` в комментариях не станет полем формы.

### Когда `=`, а когда `~`

- `=` (реальный дефолт) — для полей, где почти всегда подойдёт разумное значение
  по умолчанию и странно требовать его вручную: ссылки-заглушки (`=LINK`), счётчики
  (лайки/просмотры), служебные ID, картинки-плейсхолдеры (placehold.co).
- `~` (только подсказка) — для всего, что уникально для конкретного персонажа: имя,
  статус, биография, тексты постов и т.д. Тут дефолт по `=` выглядел бы как чужие
  забытые данные — с `~` форма стартует пустой, а подсказка только показывает формат.
- Для `|checkbox` подсказка (`~`) не имеет смысла — у галочки только два состояния,
  используй `=1` (включено по умолчанию) или просто не пиши `=` (выключено).

### Имя поля используется несколько раз

Если `{{поле}}` встречается в шаблоне несколько раз — фильтр/дефолт/подсказку нужно
указать только у ПЕРВОГО (по порядку в файле) вхождения. У остальных — просто `{{поле}}`
без параметров, значение подставится то же самое.

Поля внутри разных `{{#each группа}}` — в своём собственном пространстве имён: `{{имя}}`
внутри `{{#each посты}}` и `{{имя}}` внутри `{{#each соцсети}}` — два независимых поля.

## Как добавить новый шаблон

1. Взять готовый HTML-фрагмент (уже проверенный на реальной странице Toyhouse на все
   ограничения из `возможности_и_ограничения.md`).
2. Заменить всё, что должно быть кастомизируемым, на `{{поле...}}` по синтаксису выше.
3. Положить файл в `templates/` и добавить запись в `templates/manifest.json`:
   ```json
   { "id": "уникальный_id", "название": "Название в выпадающем списке", "файл": "имя_файла.html" }
   ```
4. Открыть `tools/index.html` — новый шаблон появится в списке при создании персонажа.

## Ограничения, которые нужно помнить при редактировании шаблона

- Никаких `<script>`, `<style>`, `<!DOCTYPE>`, `<html>/<head>/<body>` внутри шаблона —
  только фрагмент, который вставляется как есть в поле профиля/блога на Toyhouse.
- Интерактивность — только через готовые атрибуты Bootstrap (`data-toggle="tab"`,
  `data-toggle="collapse"`, `data-toggle="tooltip"`), кастомный JS не выполнится.
- Все стили — только инлайновые `style="..."` или классы Bootstrap/Font Awesome
  (они уже подключены на Toyhouse глобально).
- `gap`/`row-gap`/`column-gap` в `style=""` вырезаются сайт-санитайзером — использовать
  `margin` или классы Bootstrap (`mr-*`, `mt-*` и т.д.).
- Сложный многослойный `background:` (градиент + `url()` + `position`/`size` одной
  строкой) ломается тем же санитайзером — разбивать на отдельные `<div>` с одним
  `background`-свойством в каждом (как сделано в `character_page.html` для обложки).
- `<div>` внутри `<a>` разрывается на несколько `<a>` — использовать паттерн
  `stretched-link`: обычный `<div>` (`position:relative`) с содержимым + пустой
  `<a class="stretched-link">` в конце для клика по всей области.
